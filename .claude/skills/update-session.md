# update-session

セッション情報またはタイムテーブルを更新するときのチェックリストを実行し、必要なファイルをすべて一貫した状態に保つ。

## 対象ファイル一覧

### 1. `src/data/sessions.json` — タイムテーブルのデータソース（必須）

タイムテーブルコンポーネントが直接読み込むJSON。変更の起点となる。

各エントリのフィールド:
- `time` — 開始時刻 `"HH:MM"` 形式
- `duration` — 表示用の長さ文字列 `"30 min"` など
- `track` — `"A"` / `"B"` / `"—"` (`"—"` は全トラック共通ブロック)
- `title` / `speaker` / `tags` / `slug`
- `isBreak: true` — 休憩・転換など非セッションブロック

**注意点:**
- エントリは時刻昇順で並べること（Map の挿入順でタイムテーブルが描画される）
- 同じ `time` に Track A と Track B の両方を置く場合、common(`"—"`)エントリは置かない
- 転換ブロックは `isBreak: true` かつ `track: "—"` で追加する
- `slug` を持つエントリは対応する `content/sessions/<slug>.md` が存在しなければならない

### 2. `content/sessions/<slug>.md` — セッション詳細ページ（slug を持つエントリごと）

フロントマターと本文の両方に時刻・長さが含まれる場合がある。

チェック項目:
- `time:` フィールドが `sessions.json` の `time` と一致しているか
- `duration:` フィールドが `sessions.json` の `duration` と一致しているか
- 本文中に `"HH:MM 枠"` のような時刻への言及があれば更新する

### 3. `src/components/Timetable.tsx` — 描画コンポーネント（構造変更時のみ）

通常の時刻・長さ変更では触らなくてよい。以下の場合のみ確認する:
- トラック数の増減
- サマリーカード内の `3 sessions · 50min each` などのハードコード文言がずれた場合

## 作業手順

```
1. sessions.json を読み込み、変更対象のエントリを特定する
2. sessions.json を編集する
   - time / duration を更新
   - エントリ順が時刻昇順になっているか確認・修正
   - 転換ブロックの追加・削除が必要なら対応
3. 変更したエントリの slug を列挙する
4. 各 slug に対応する content/sessions/<slug>.md を開く
   - フロントマターの time / duration を sessions.json と揃える
   - 本文中の時刻言及を grep で探して更新する
5. Timetable.tsx のサマリーカードのハードコード文言が実態とかけ離れていないか確認する
6. 変更ファイルをまとめてコミットする
```

## よくある漏れパターン

| 漏れ | 症状 |
|---|---|
| sessions.json だけ更新して .md を更新しない | 詳細ページに古い時刻が残る |
| .md のフロントマターは更新したが本文の `"15:10 枠です"` を直し忘れる | 詳細ページ本文に古い時刻が残る |
| 転換ブロックを追加せず sessions.json の順序も崩れる | タイムテーブルの並び順が乱れる |
| slug を変えずに time だけ変えて OK と思う | slug はファイル名・URLになるので基本変えない。time と slug のずれは許容 |

## セッションを新規追加するときの手順

1. `sessions.json` に新エントリを追加（時刻昇順の位置に挿入）
2. `content/sessions/<slug>.md` を新規作成（既存ファイルをテンプレートとして複製）
3. 詳細ページのフロントマターと本文を正しく設定する

## セッションを削除するときの手順

1. `sessions.json` からエントリを削除
2. 対応する `content/sessions/<slug>.md` を削除（またはリダイレクト設定）
3. 前後の転換ブロックが不要になっていないか確認
